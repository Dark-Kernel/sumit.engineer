systemd service dependencies with After= vs Requires=

2025-01-29
Discovered the subtle difference: After= only controls startup order, while Requires= creates a dependency that stops this service if the required service fails. Use Wants= for loose coupling.
# Start after network, but don't fail if network fails After=network.target Wants=network.target # vs requiring PostgreSQL to be running Requires=postgresql.service After=postgresql.service

Docker multi-stage builds can share intermediate layers

2025-01-28
Using --target flag allows building specific stages, and you can reference previous stages with COPY --from=stage_name. Great for separating build tools from runtime.
FROM node:18 AS builder WORKDIR /app COPY package*.json ./ RUN npm ci --only=production FROM node:18-alpine AS runtime COPY --from=builder /app/node_modules ./node_modules

PostgreSQL UPSERT with ON CONFLICT in Python

2025-01-27
PostgreSQL's ON CONFLICT clause is perfect for idempotent operations. Much cleaner than try/except INSERT/UPDATE patterns.
INSERT INTO users (email, name) VALUES (%s, %s) ON CONFLICT (email) DO UPDATE SET name = EXCLUDED.name, updated_at = NOW()

ss command is the modern replacement for netstat

2025-01-26
ss is faster and provides more detailed socket information. Use ss -tlnp to show listening TCP ports with process info.
# Show listening TCP ports with process names ss -tlnp # Show established connections to specific port ss -t state established '( dport = :80 or sport = :80 )'

Nginx rate limiting with burst and nodelay

2025-01-25
The burst parameter allows temporary spikes, while nodelay processes burst requests immediately instead of queuing them.
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s; location /api/ { limit_req zone=api burst=20 nodelay; proxy_pass http://backend; }

journalctl can follow logs in real-time like tail -f

2025-01-24
Use journalctl -f to follow logs, -u service_name for specific services, and --since "1 hour ago" for time-based filtering.
# Follow nginx logs from the last hour journalctl -u nginx -f --since "1 hour ago" # Show logs with specific priority journalctl -p err -f
